### SECONDARY (best-effort): indicative Columbia-subset F1 â€” **SKIPPED (honest)**

Attempted but not run, for a concrete data/methodology reason (not a model bug):

- The Columbia eval (`~/LR-ASD/Columbia_test.py --evalCol`, with `yt-dlp`
  substituted for the dead `youtube-dl`) matches predictions to ground-truth by
  **absolute frame index** from a frame-0 full-video extraction.
- The earliest GT label is **absolute frame 25000 (~16.7 min in)**; labels span
  ~frames 25000â€“155880 (the back two-thirds of the 87-min / 5212 s video).
- Therefore any short `--duration` subset that fits the one-turn / 6 GB budget
  contains **zero** labeled frames. Reaching even the first label window means
  processing ~23 min of footage (~35k S3FD detections) â€” the multi-hour
  full-scale run this turn explicitly scoped out.
- A hand-rolled offset-remapped subset *could* shorten it, but a frame-offset bug
  would yield an **unvalidated** F1 â€” worse than an honest skip under the
  no-faking rail. The bit-identical equivalence above already transfers the
  published 96.4% / 94% numbers, so the indicative subset is not the must-have.
