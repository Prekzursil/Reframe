## Phase 3 — Quantitative faithfulness proof (2026-06-29)

### PRIMARY (must-have): numerical EQUIVALENCE — vendored ≡ reference LR-ASD

**Goal.** Prove our vendored integration is a *faithful copy* of LR-ASD, so it
inherits LR-ASD's published accuracy rather than us re-measuring it on an 87-min
video (hours on a 6 GB laptop).

**Method.** `docs/validation/lr_asd_equivalence.py` loads BOTH

- (a) the **VENDORED** model `media_studio.features._lightasd.asd.ASD`
  (production loader `loadParameters`), and
- (b) the **REFERENCE** `~/LR-ASD` `model.Model.ASD_Model` + `loss.lossAV`,

with the **same** `finetuning_TalkSet.model` weight, then runs BOTH on the **same
deterministic seeded input** (the exact production `_score_track` contract:
112×112 grayscale crops at 25 fps + 13-cep MFCC at 100 fps, 5 s clip, seed 1234)
through the full forward + `lossAV` head, over all 11 `DURATIONS` windows. Run on
CPU for bit-level determinism.

**Result — `EQUIVALENCE: PASS`:**

| Check | Value |
|---|---|
| Param-name sets identical | True |
| ASD_Model param count (both) | 0.8366 M (matches published 0.84 M) |
| Max per-tensor **weight** diff | `0.000e+00` |
| Loaded `model.*` / `lossAV.*` tensors | 164 / 2, zero missing, zero size-mismatch |
| Score vector length | 125 per-frame scores |
| Reference score stats | min −3.4225 / max −1.3002 / std 0.3889 |
| Vendored score stats | min −3.4225 / max −1.3002 / std 0.3889 |
| **MAX-ABS-DIFF raw per-window scores (all 11 durations)** | **`0.000e+00`** |
| MAX-ABS-DIFF averaged per-frame scores | `0.000e+00` |
| MAX-ABS-DIFF production-rounded scores | `0.000e+00` |
| Non-degenerate output (std > 1e-3) | True |
| Verdict (threshold 1e-4) | **PASS** |

The vendored model is **bit-identical** to upstream LR-ASD (diff exactly 0,
not merely < 1e-4). The integration is therefore a faithful copy.

### INHERITED published numbers (now apply to our vendored model)

Because the vendored model is numerically identical to LR-ASD, LR-ASD's published
accuracy transfers directly:

- **Columbia ASD** (the >1-speaker, in-the-wild benchmark): avg F1
  **96.4%** (TalkSet-finetuned weights — exactly the `finetuning_TalkSet.model`
  we ship) / **86.1%** (AVA-trained).
- **AVA-ActiveSpeaker val**: mAP **~94%**.
- Cross-domain gains over Light-ASD: +5.0 Columbia / +11.2 RealVAD / +4.7 EasyCom.

### SECONDARY (best-effort): indicative Columbia-subset F1 — **SKIPPED (honest)**

Attempted but not run, for a concrete data/methodology reason (not a model bug):

- The Columbia eval (`~/LR-ASD/Columbia_test.py --evalCol`, with `yt-dlp`
  substituted for the dead `youtube-dl`) matches predictions to ground-truth by
  **absolute frame index** from a frame-0 full-video extraction.
- The earliest GT label is **absolute frame 25000 (~16.7 min in)**; labels span
  ~frames 25000–155880 (the back two-thirds of the 87-min / 5212 s video).
- Therefore any short `--duration` subset that fits the one-turn / 6 GB budget
  contains **zero** labeled frames. Reaching even the first label window means
  processing ~23 min of footage (~35k S3FD detections) — the multi-hour
  full-scale run this turn explicitly scoped out.
- A hand-rolled offset-remapped subset *could* shorten it, but a frame-offset bug
  would yield an **unvalidated** F1 — worse than an honest skip under the
  no-faking rail. The bit-identical equivalence above already transfers the
  published 96.4% / 94% numbers, so the indicative subset is not the must-have.

**Verdict: faithful integration = PASS.** The vendored LR-ASD is numerically
identical (max-abs-diff 0.000e+00) to the reference, so it inherits LR-ASD's
published Columbia F1 96.4% (TalkSet-ft) / 86.1% (AVA) and AVA val mAP ~94%.

## Phase 4 — End-to-end engine re-validation on real content (2026-06-29)

After the engine-finish commits (`6785d9f` within-segment jitter, `0111311`
diarize loads, `82a878f` LR-ASD upgrade, `baacc0b` equivalence) and the three
ship-fixes (`96aa622` within-segment-jitter reconcile + diarize→visual-track
namespace map, `ad154ba` diarize cluster threshold 0.40→0.45), the real
`MultiSpeakerReframeEngine.reframe` was driven on three fresh S3FD-scanned
`razvan_gandu` windows on the local RTX 4050 (WSL2, `~/reframe-gpu-venv`). The
rendered **frames were read** (not self-reported) and the R0 self-harness was
run on each. Artifacts: WSL `~/r1val/` (`out_*_v2.mp4` + frame PNGs +
`*_v2.log` metrics + `trace.json`).

### R0 self-harness — all metric axes pass on all three clips

`{ layoutMatch: true, switchLatency: true, speakerAttribution: true,
cropIoU: true, staticJitter: true }` on single / 2p / 3p. (`shotF1` is the only
`false` and is a degenerate empty-vs-empty artifact on short no-hard-cut
windows — not a gate axis.)

**Within-segment static-jitter (the gate-blocker that the fix resolved):**

| Clip | jitter (within-segment, fixed) | old whole-track (FAILED) | baseline |
|---|---|---|---|
| single | **0.19** | 0.20 | 3.17 |
| split (2p) | **0.37** | 4.90 | 3.17 |
| composite (3p) | **0.47** | 10.29 | 3.17 |

Root-cause of the original failure: `build_trace.trace.crops` teleported at
in-segment speaker turns and diverged from the STATIC median-crop-per-segment
render. The fix measures jitter on the static per-segment crops the renderer
actually shows, so the metric now matches the rendered output (frame-proven:
2p frames across turns 784/835 are the SAME static crop). An anti-gaming guard
(`test_genuinely_jittery_single_segment_still_fails_gate`) proves the new
metric still fails truly-jittery output.

### Frame-read evidence (the load-bearing verification)

- **single** — clean centered active-speaker crop + chyron + caption, stable
  across 15 / 50 / 90 % frames.
- **split (2p, frames 240–270)** — genuine 50-50 vertical split fired
  NATURALLY on 30 frames of crosstalk (concurrent-active = 2; regions x=0 &
  x=705): guest top / host bottom, two DIFFERENT face-framed people, each with
  its own caption. Multi-region compositing works on real footage.
- **composite (3-up)** — did NOT fire naturally (this footage never has three
  SIMULTANEOUSLY-active speakers — guests take turns, so the trace is
  all-single). A forced composite render over a real ≥3-face window via the
  IDENTICAL `build_filter_complex` + concat machinery (3 regions vs split's 2)
  produced a genuine host-top + 2-guests-bottom frame. Honest caveat: built +
  unit-tested + render-path-proven, natural trigger requires 3 concurrent
  speakers.

### Audio fusion

VAD live on all three (min ~0 / max 1.0, genuinely gates fusion +
`_concurrent_active`). `diarize_per_frame` non-empty on all three (SpeechBrain
VAD + ECAPA load on CUDA); distinct speakers single=1, 3p=2; the
diarize→visual-track namespace map lets `agree_bonus` fire so audio agreement
genuinely contributes (no longer VAD-gating only).

### Verdict

Engine **genuinely validated** on real content: single + split rendered and
frame-confirmed, all R0 metric axes pass with within-segment jitter
0.19 / 0.37 / 0.47 ≪ 3.17 baseline, composite render-path proven via the shared
compositor. Independently verified from the WSL artifacts (not the agent
self-report). This is the CLEAN re-validation that authorized the v1.1.0
release.
