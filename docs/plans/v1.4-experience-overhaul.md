# Reframe v1.4 — Experience Overhaul (plan)

Grounded in a 5-lens design exploration (wf_1ed43a8c) over the live screenshots + code. Locked decisions:
G-1 UX overhaul · auto-update = GitHub feed (live at v1.3.0) · one big v1.4 · **install options = in-app
first-run screen** (NSIS stays a clean assisted install) · **visual mood = cool blue-gray** · keep-a-copy =
opt-in w/ free-space preflight (user's explicit request).

KEY INSIGHT: most machinery already EXISTS; the work is wiring + making it visible, not new capability.
`main.ts` already broadcasts `bootstrap.progress`; `bootstrap.py` already takes `--assets` (core/default tiers);
the B3 playback-proxy already fixes code-4; the sidecar already has reveal_source/relink/lineage. Coverage gate:
`.coverage-thresholds.json` = 100% (renderer vitest, sidecar pytest). Ships via in-place auto-update.

## Phase 0 — SAFETY (RED-TEAM; MUST land before any v1.4 auto-update or it can BRICK a working install)
- WU-S1 single-instance lock: `app.requestSingleInstanceLock()` in main.ts (focus existing window on 2nd
  launch) — else two bootstraps pip into the same env + open the same library.db = corruption.
- WU-S2 version-aware re-bootstrap: gate first-run on a persisted shipped-requirements FINGERPRINT, not just
  marker existence — else an update that adds Python deps starts the sidecar against a stale env ("No module X").

## Phase 1 — Kill the loud pains (mostly wiring existing machinery)

  "sidecar crashed").

  + its mount-time RPCs; shows phase + progress bar (fed by existing "assets NN.N%"/"step k/2" lines);
  auto-transitions when sidecar status = running. (Kills "sidecar is not running" on first launch.)
- WU-1c in-app install profiles (Minimum/Default/Full/Custom + what/why + approx size) -> route `--assets` into
  `runFirstRunBootstrap` (spawns with NO args today); profile->asset map synced w/ bootstrap.py
  core_first_run_assets (keep YuNet/S3FD/LR-ASD CORE FLOOR in every level — else silent centre-crop).
- WU-1d surface FirstRunChooser (local-vs-cloud) into the first-run flow (it's buried in Settings today).
- WU-1e Player preview fix: sidecar-state-aware (503/building -> "Building preview…" not raw code-4; reload on
  `proxy.state` ready; loud reason on real error) + fix fixed-height/overflow CSS ("halfway cut at top").
- WU-1f Library provenance: source-path reveal + "on disk / MISSING" badge + open-in-folder + hash-verified
